1. MVC 패턴 
    MVC 패턴은 Model, View, Controller의 약자로 소프트웨어 디자인패턴 중 하나입니다.
        - Model : DATA, 정보들의 가공을 책임지는 컴포넌트(DB)
        - View : 사용자 인터페이스 요소로, 데이타를 기반으로 사용자들이 볼 수 있는 화면
        - Controller : 모델(+ 로직)과 뷰를 연결해주는 다리 역할
